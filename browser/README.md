# Irogen Browser

Web browser version of Irogen using WebAssembly.

## Features

- 🌐 Browser-based terminal access
- 🔗 Connect to remote terminal sessions
- 📱 Mobile-friendly interface
- ⚡ Native WebAssembly performance
- 🎨 Dark theme terminal interface

## Building

### Prerequisites

- Rust (latest stable)
- `wasm-pack` for WebAssembly compilation
- Node.js (for development server)

### Build Commands

```bash
# Install wasm-pack if not already installed
cargo install wasm-pack

# Build for web target
npm run build

# Development build (with debug info)
npm run dev

# Start local development server
npm run serve
```

## Usage

1. **Build the project**:

   ```bash
   npm run build
   ```

2. **Start a local server**:

   ```bash
   npm run serve
   ```

3. **Open your browser**:
   Navigate to `http://localhost:8080`

4. **Connect to a session**:
   - Get a session ticket from a Irogen host
   - Enter the ticket in the session ticket field
   - Click "Connect"

5. **Create a terminal**:
   - Once connected, click "Create Terminal"
   - Start typing commands in the input field

## Session Tickets

Session tickets are generated by Irogen hosts and encode connection information.

To get a ticket:

1. Start a Irogen host: `./cli host`
2. The host will print a session ticket
3. Copy this ticket and use it in the browser client

### API Reference

#### TerminalNode

```typescript
const node = new TerminalNode();
await node.spawn();
const nodeId = node.node_id();
const session = await node.create_session(ticket);
```

#### TerminalSession

```typescript
const sessionId = session.session_id();
const receiver = session.receiver;
await session.send_input(terminalId, "ls -la");
const terminalId = await session.create_terminal(config);
```

#### Configuration

```typescript
const config = {
  name: "my-terminal",
  shell_path: "/bin/bash",
  working_dir: "/home/user",
  rows: 24,
  cols: 80,
};
```

## Browser Compatibility

- Chrome 67+
- Firefox 63+
- Safari 15+
- Edge 79+

## Limitations

- File transfer not yet supported in browser
- Limited terminal capabilities compared to desktop app
- Requires WebAssembly and modern browser features

## Security

- All communication uses the same end-to-end encryption as desktop clients
- Session tickets contain connection information only
- No sensitive data is stored in the browser

## Troubleshooting

### Build Issues

```bash
# Clear build cache
npm run clean
npm run build
```

### Runtime Issues

1. **WASM loading failed**: Make sure your browser supports WebAssembly
2. **Connection failed**: Check the session ticket is valid and not expired
3. **No output**: Make sure the host is running and accessible

### Debug Console

Open browser developer tools to see debug output from the WASM module.
